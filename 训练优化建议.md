# 3D Gaussian Splatting 训练优化建议

## 核心观点：更多训练迭代 = 更好结果

您观察到的"不透明区域过多"问题，很大程度上是因为**30,000次迭代对于Tanks&Temples这样的复杂场景可能不够**。

## 训练迭代与质量关系

### 标准训练计划
```
阶段        迭代次数     主要目标
────────────────────────────────────────────
初始化      0-1,000     从COLMAP点云初始化
稠化阶段    1,000-15,000 增加高斯点密度，学习几何
优化阶段    15,000-30,000 优化颜色、不透明度、缩放
精炼阶段    30,000-70,000 精细调整，减少伪影
```

### 不同迭代次数的预期效果

#### 30,000次迭代（当前状态）
- ✅ 基础几何学习完成
- ✅ 颜色大致正确
- ❌ 不透明度未充分优化
- ❌ 仍有大量伪影和噪声
- ❌ 细节不够清晰

#### 50,000-70,000次迭代（推荐）
- ✅ 几何更精确
- ✅ 不透明度分布更合理
- ✅ 减少阴影和不透明斑块
- ✅ 颜色更准确
- ✅ 细节更清晰

#### 100,000+次迭代（高质量）
- ✅ 接近最优质量
- ✅ 最小化伪影
- ✅ 最佳不透明度分布
- ✅ 最清晰的细节
- ⚠️ 训练时间显著增加

## 关键训练参数优化

### 1. 稠化参数（影响几何质量）
```bash
--densify_from_iter 500        # 更早开始稠化
--densify_until_iter 20000     # 延长稠化阶段
--densify_grad_threshold 0.0001 # 更低的梯度阈值，更积极稠化
--densification_interval 100   # 更频繁检查稠化
```

### 2. 不透明度优化参数
```bash
--opacity_reset_interval 2000  # 更频繁重置不透明度
--opacity_init 0.1             # 初始不透明度
--opacity_min 0.005            # 最小不透明度
```

### 3. 学习率调整
```bash
--position_lr_init 0.00016
--position_lr_final 0.0000016  # 更低的最终学习率
--position_lr_delay_mult 0.01
--position_lr_max_steps 70000  # 匹配总迭代次数
```

### 4. 球谐函数优化
```bash
--sh_degree 3                  # 使用3阶球谐函数
--feature_lr 0.0025
--sh_lr 0.0025
```

## 总结

### 关键结论
1. **更多迭代 = 更好质量**：这是3D Gaussian Splatting的核心原则
